# Study-Board 환경별 관리 방법

## 프로젝트 구조

```
Study-Board/
├── back/          # NestJS 백엔드 (포트 8888)
│   ├── .env       # 백엔드 환경변수 (git에 올리지 않음)
│   ├── upload/           # 게시글 이미지
│   ├── userUpload/       # 프로필 이미지
│   ├── channelUpload/    # 채널 이미지
│   ├── suggestionUpload/ # 건의사항 이미지
│   └── videoUpload/      # 동영상
├── front/         # Next.js 프론트엔드 (포트 3000)
│   ├── .env       # 프론트엔드 환경변수
│   └── next.config.mjs
```

## 2. EC2 배포 환경

### 백엔드 `.env`
```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=<EC2 DB 비밀번호>

JWT_SECRET=<강력한 시크릿 키>
AWS_REGION=ap-northeast-1
AWS_S3_BUCKET=park-board
CORS_ORIGINS=http://park-aws-study.com,https://park-aws-study.com,http://13.158.115.186,https://13.158.115.186
PORT=8888
NODE_ENV=production
DB_SYNC=false
```

### 프론트엔드 `.env`
```env
NEXT_PUBLIC_BASE_URL=http://park-aws-study.com:8888
NEXTAUTH_SECRET=<강력한 시크릿 키>
AUTH_SECRET=<강력한 시크릿 키>

```

### EC2 빌드 및 실행
```bash

# 프론트엔드

npm install

npm run start        # next start (포트 3000)
```

## 3. 환경별 차이점 요약

| 항목 | 로컬 | EC2 |
|------|------|-----|
| `DB_NAME` | `board-study` | `park-board` |
| `DB_PASSWORD` | `6429` | EC2 DB 비밀번호 |
| `JWT_SECRET` | `park` | 강력한 키 사용 |
| `CORS_ORIGINS` | `localhost:3000` | `park-aws-study.com` 등 |
| `NEXT_PUBLIC_BASE_URL` | `http://localhost:8888` | `http://park-aws-study.com:8888` |

| 백엔드 실행 | `npm run start:dev` | `npm run start:prod` |
| 프론트엔드 실행 | `npm run dev` | `npm run build && npm run start` |

## 4. 코드에서 하드코딩 주의할 곳

### WebSocket 연결 URL
`front/src/app/utils/websocket.ts` 68번째 줄, 375번째 줄에 WebSocket URL이 하드코딩되어 있음.

```typescript
// 현재 상태 (하드코딩)
this.socket = io("http://localhost:8888", { ... });
```

EC2 배포 시에는 이 값을 환경변수로 바꾸거나, 배포 전에 수동으로 변경해야 함.

```typescript
// 개선안: 환경변수 사용
this.socket = io(process.env.NEXT_PUBLIC_BASE_URL || "http://localhost:8888", { ... });
```

### next.config.mjs rewrite 규칙
`front/next.config.mjs`에서 `/upload/:slug`와 `/videoUpload/:slug`만 rewrite 설정이 있음.
`NEXT_PUBLIC_BASE_URL` 환경변수를 참조하므로 `.env`만 바꾸면 자동 반영됨.

## 6. 주의사항

1. **`.env` 파일은 절대 git에 커밋하지 않는다** - `.gitignore`에 포함되어 있는지 확인
2. **`DB_SYNC=false`** - EC2 운영환경에서는 반드시 설정. 안 하면 엔티티 변경 시 테이블이 자동으로 바뀌어 데이터 유실 위험
3. **JWT_SECRET** - 로컬과 EC2에서 다른 값을 쓰면 로컬에서 발급한 토큰이 EC2에서 안 먹힘 (의도된 것)
4. **CORS_ORIGINS** - EC2에서는 실제 도메인/IP를 모두 나열해야 함. 빠뜨리면 API 호출이 차단됨
5. **이미지 업로드 폴더** - EC2에서 폴더가 없으면 자동 생성이 안 될 수 있으므로 미리 mkdir 해둘 것
