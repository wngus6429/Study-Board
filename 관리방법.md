# Study-Board 환경별 관리 방법

## 프로젝트 구조

```text
Study-Board/
├── back/          # NestJS 백엔드 (포트 8888)
│   ├── .env       # 백엔드 환경변수 (git에 올리지 않음)
│   └── src/common/utils/multer.options.ts # 환경에 따라 로컬/S3 스토리지 분기
├── front/         # Next.js 프론트엔드 (포트 3000)
│   ├── .env       # 프론트엔드 환경변수
│   └── next.config.mjs # S3 도메인(awsamazonaws.com) 허용 및 프록시 설정
```

---

## 1. 로컬 개발 환경

### 백엔드 `.env` (`back/.env`)
```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_local_password
DB_NAME=board-study
JWT_SECRET=your_secret_key
CORS_ORIGINS=http://localhost:3000,http://127.0.0.1:3000
NODE_ENV=development
```
*로컬 환경에서는 `NODE_ENV=development` 또는 미설정 상태일 경우, 이미지가 `upload/` 등 백엔드 로컬 디렉토리에 저장됩니다.*

### 프론트엔드 `.env` (`front/.env`)
```env
NEXT_PUBLIC_BASE_URL=http://localhost:8888
NEXTAUTH_SECRET=park
AUTH_SECRET=park
NEXT_PUBLIC_SITE_PASSWORD=8812
```

### 실행
```bash
# 백엔드
cd back
npm run start:dev

# 프론트엔드
cd front
npm run dev
```

## 2. AWS 배포 환경 (RDS + S3 + 2 EC2)

아키텍처 구성:
- **Database**: Amazon RDS (MySQL)
- **File Storage**: Amazon S3 (이미지 및 비디오)
- **Frontend**: EC2 인스턴스 (Nginx + Next.js 또는 Nginx 리버스 프록시)
- **Backend**: EC2 인스턴스 (NestJS)

### 백엔드 `.env` (`back/.env`)
```env
# RDS 연결 정보
DB_HOST=your-rds-endpoint.ap-northeast-2.rds.amazonaws.com
DB_PORT=3306
DB_USER=admin
DB_PASSWORD=your_rds_password
DB_NAME=park-board
DB_SYNC=false

# 기타 설정
JWT_SECRET=<강력한 시크릿 키>
CORS_ORIGINS=http://your-frontend-domain.com,https://your-frontend-domain.com
PORT=8888

# S3 파일 업로드 활성화 및 자격 증명
NODE_ENV=production
AWS_REGION=ap-northeast-2
AWS_ACCESS_KEY_ID=your_aws_access_key
AWS_SECRET_ACCESS_KEY=your_aws_secret_key
S3_BUCKET_NAME=your_s3_bucket_name
```
*`NODE_ENV=production` 으로 설정되어야 이미지가 S3 버킷으로 업로드되며, DB에 URL 전체 경로가 저장됩니다.*

### 프론트엔드 `.env` (`front/.env`)
```env
NEXT_PUBLIC_BASE_URL=https://api.your-domain.com  # 백엔드 도메인 또는 ALB 주소
NEXTAUTH_SECRET=<강력한 시크릿 키>
AUTH_SECRET=<강력한 시크릿 키>
NEXT_PUBLIC_SITE_PASSWORD=8812
NEXTAUTH_URL=https://your-frontend-domain.com
```

### EC2 배포 및 실행 가이드

#### 1) 백엔드 EC2 (NestJS)
```bash
# PM2 설치 및 빌드
npm install -g pm2
cd back
npm install
npm run build

# PM2로 백엔드 실행
pm2 start dist/main.js --name study-board-back
pm2 startup
pm2 save
```

#### 2) 프론트엔드 EC2 (Nginx + Next.js)
```bash
# PM2 설치 및 빌드
npm install -g pm2
cd front
npm install
npm run build

# PM2로 프론트엔드 실행 (포트 3000)
pm2 start npm --name study-board-front -- start
pm2 startup
pm2 save
```

**Nginx 설정 (`/etc/nginx/sites-available/default`)**
프론트엔드 EC2에서 Nginx를 사용해 80/443 포트를 3000번 포트로 리버스 프록시합니다.
```nginx
server {
    listen 80;
    server_name your-frontend-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 3. 환경별 차이점 요약

| 항목 | 로컬 환경 | AWS 배포 환경 |
|------|-----------|--------------|
| **DB 위치** | 로컬 MySQL (localhost) | Amazon RDS |
| **파일 스토리지** | 백엔드 내 폴더 (upload/ 등) | Amazon S3 |
| **파일 URL 구조** | `/upload/파일명.jpg` | `https://s3.amazonaws.com/...` |
| **DB_SYNC** | `true` (기본값) | `false` (운영 DB 스키마 보호) |
| **NODE_ENV** | `development` | `production` (S3 업로드 트리거) |
| **프론트 웹서버** | Next.js 내장 서버 (3000) | Nginx 리버스 프록시 -> Next.js (3000) |

## 4. 코드에서 하드코딩 주의할 곳

### WebSocket 연결 URL
`front/src/app/utils/websocket.ts` 파일 등에 WebSocket URL이 하드코딩되어 있다면, 환경변수화 해야 합니다.
```typescript
// 개선안: 환경변수 사용
this.socket = io(process.env.NEXT_PUBLIC_BASE_URL || "http://localhost:8888", { ... });
```

## 5. AWS 배포 체크리스트

- [ ] **RDS**: 퍼블릭 엑세스 제한 확인 및 백엔드 EC2 보안그룹(SG)에서만 3306 포트 접근 허용 설정
- [ ] **S3**: 버킷 생성 및 IAM 사용자 정책 할당 (s3:PutObject, s3:GetObject, s3:DeleteObject 권한 부여). 객체 퍼블릭 접근 허용.
- [ ] **백엔드 EC2**: `.env`에 RDS 주소와 S3 Key 설정, `NODE_ENV=production` 입력 확인. 백엔드 포트(8888)에 대해 프론트엔드 EC2 또는 ALB의 인바운드 허용.
- [ ] **프론트엔드 EC2**: Nginx 설치 및 리버스 프록시 설정 (3000번 포트 연결), `.env`에 백엔드 API 주소 올바르게 입력.
- [ ] **프론트엔드 next.config.mjs**: S3 이미지 도메인이 `remotePatterns`에 등록되어 있는지 확인 (완료됨)
- [ ] **공통**: 각 서버에 PM2로 애플리케이션 데몬화 및 `pm2 startup` 등록 완료.

## 6. 주의사항

1. **`.env` 파일은 절대 git에 커밋하지 않는다** - `.gitignore`에 포함되어 있는지 확인.
2. **S3 자격증명 노출 주의** - AWS Access Key는 절대 소스코드 내 하드코딩하지 않고 `.env`로 관리할 것.
3. **CORS_ORIGINS** - EC2 배포 시 프론트엔드의 정확한 도메인을 백엔드 `.env`에 나열해야 합니다.
