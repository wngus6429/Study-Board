# AI 시대에 사람이 알아야 할 공부 우선순위

> **작성일**: 2026년 2월 20일  
> **목적**: AI가 코딩을 많이 대체하는 시대에, 사람이 집중해야 할 학습 방향 정리

---

## 🤖 AI가 이미 잘 해주는 것 (상대적으로 덜 중요해짐)

- **Swagger/OpenAPI 문서화** → AI가 코드 보고 거의 자동으로 작성 가능
- **CRUD 기본 패턴** → 거의 자동화됨
- **기본적인 테스트 코드 뼈대** → AI가 spec 파일 초안 금방 생성

---

## 🧠 사람이 알아야 할 — 미래 유망한 공부

### 🥇 1순위: 아키텍처 / 설계 판단력

> AI는 "어떻게 짜는지"는 알지만, **"왜 이렇게 설계해야 하는지"** 판단은 아직 약함

- `story.service.ts` (74KB) 같은 걸 **언제, 어떻게 나눌지** 판단
- Redis를 **언제 쓰고 언제 안 쓰는지** 트레이드오프 이해
- DB 마이그레이션 전략 — 서비스 중단 없이 스키마 변경하는 방법
- **→ 이게 시니어와 주니어를 가르는 가장 큰 차이**

---

### 🥈 2순위: 인프라 / DevOps 이해

> AI가 코드를 짜도, **어디서 어떻게 돌아가는지**는 사람이 알아야 배포 가능

- Docker + docker-compose 직접 구성 경험
- AWS 설계 이해 (프로젝트 내 `AWS_ARCHITECTURE_DIAGRAM.md` 직접 분석)
- CI/CD 파이프라인 구성 원리 이해
- **→ AI가 코드를 생성해도, 배포/운영은 여전히 사람의 영역**

---

### 🥉 3순위: 테스트 전략 설계

> "테스트 코드 짜줘"는 AI가 하지만, **무엇을 어떻게 테스트할지 전략**은 사람이 결정

- 무엇이 핵심 비즈니스 로직인지 파악하는 능력
- Integration Test vs Unit Test 중 어디에 집중할지 판단
- 테스트가 없을 때 실제로 어떤 문제가 생기는지 직접 경험

---

## 📌 결론 — 지금 당장 공부 우선순위

```
🔥 지금 바로 가치 있는 것
   → Docker 컨테이너화 실습            (인프라 이해)
   → story.service.ts 리팩터링 연습    (설계 판단력)
   → TypeORM 마이그레이션 실습          (DB 운영 이해)

📈 중장기적으로 가치 있는 것
   → Redis 캐싱 레이어 추가             (시스템 설계)
   → 테스트 전략 수립 + 작성            (품질 판단력)

⬇️ 상대적으로 덜 긴급한 것
   → Swagger 완성, 기본 Guard 구현      (AI가 도와줄 수 있음)
```

## 💡 핵심 한 줄 요약

> 공부하는 게 AI 시대엔 훨씬 가치 있다.
